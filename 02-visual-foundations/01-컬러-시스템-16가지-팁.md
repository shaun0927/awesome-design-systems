# 디자인 시스템의 컬러: 16가지 핵심 팁 (Nathan Curtis)

## 📌 핵심 개념

컬러는 단순히 "예쁜 색 고르기"가 아니라, **확장 가능하고 접근 가능하며 일관된 시스템을 구축하는 과정**

### 4가지 영역
1. **Primary Palette**: 브랜드 컬러, 중립색, 인터랙티브 블루
2. **Tints & Shades**: 각 컬러의 밝기 범위 설정
3. **Secondary Palettes**: 피드백 컬러, 테마 컬러
4. **Contrast & Accessibility**: WCAG 2.0 준수

## 🎯 16가지 컬러 팁

### Primary Palette

#### #1. 브랜드 컬러를 빠르게 확정하라
- 핵심 브랜드 컬러 1~3개를 초기에 결정
- 변경 시 시스템 전체에 파급 효과

#### #2. 브랜드 컬러 변경 시 브랜드팀 참여시켜라
- 브랜드 컬러 = 브랜드팀 영역
- 조정 전 반드시 협의

#### #3. 중간 중립색을 버려라
```
밝은 그레이 (05, 10, 20, 30) ✅
중간 그레이 (40~60) ❌ 접근성 위험
어두운 그레이 (70, 80, 90) ✅
```

#### #4. "디지털 블루"를 채택하라
- 링크는 태초부터 파란색
- 채도 높은 파란색을 유틸리티 컬러로

### Tints & Shades

#### #5. 명암 단계를 스택으로 시각화하라
```
INDIGO
050  #E8EAF6
100  #C5CAE9
200  #9FA8DA
500  #3F51B5  ← Base
900  #1A237E
```

#### #6. Lightness 기준으로 명명하라
```scss
// 올바른 방식
$color-gray-05   // HSL Lightness 기반
$color-gray-10
$color-gray-20   // 새로 추가: $color-gray-15 가능
```

#### #7. 명암 단계 수량을 제한하라
- 대부분 시스템: 4~6단계면 충분
- Material Design(10단계)은 무수한 제품용

#### #8. 색상 변형 방법을 명시하라
- **Handpicked**: 디자이너가 직접 선택
- **Functional Transform**: `darken(10%)` - 신중히 사용

### Secondary Palettes

#### #9. 피드백 컬러 세트를 정의하라
```
SUCCESS (녹색)
WARNING (노란색)
ERROR (빨간색)
INFORMATIONAL (파란색)
```

#### #10. 테마 다양성을 보여라
- Marriott 예시: Courtyard, Renaissance, JW Marriott 등

#### #11. 테마 작동 방식을 정의하라
```
테마 적용 가능: 버튼 background, 활성 탭 background
테마 적용 금지: 긴 텍스트, 링크 색상 (가독성 문제)
```

#### #12. 컬러 믹싱 가이드는 나중에 (최소한)
- 대부분 시스템 = 그런 유연성 불필요
- 선택을 전파하는 도구 제공 (선택 자체는 사용자)

### Contrast & Accessibility

#### #13. 대비를 조기에 반복적으로 점검하라
- WCAG 2.0 규칙 숙지 (3.0 / 4.5)
- Tanaguru 같은 도구 상시 활용

#### #14. 접근 가능한 색상 선택 범위를 탐색하라
```
대화 변화:
전: "실패했네, 다시 시도"
후: "블루가 이 정도로 어두워야 하는구나"
```

#### #15. 반전 조합 해결하라
- Light on Dark / Dark on Light
- SASS 함수로 대비 매트릭스 자동 계산

#### #16. 컬러로 접근성 인식을 확산하라
- 컬러 = 시스템의 핵심
- 접근성 대비 = 컬러의 핵심
- 브랜드 매니저, 디자인 리드, 개발자, 임원 모두 주목
