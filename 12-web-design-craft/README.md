## Key Takeaways

- **The golden rule: "Don't make me think"**: Users should instantly understand what to do without cognitive effort. If they pause to wonder "What does this do?", you've already failed. Design for immediate comprehension, not clever innovation.
- **Use HSL for maintainable color systems**: HSL (Hue, Saturation, Lightness) makes color manipulation trivialβ€”adjust lightness for hover states, reduce saturation for disabled states. Far more intuitive than RGB for systematic color variations.
- **Build component-first, not page-first**: Think like LEGO blocks. Create reusable components (buttons, cards, forms) first, then assemble pages. Use design systems (Tailwind, Bootstrap) to avoid writing styles from scratch every time.
- **Responsive = box rearrangement**: Every element is a box. Responsive design means reorganizing boxes into different rows/columns based on screen size. Master flexbox and grid, understand parent-child relationships, and always ask "How many columns?" first.
- **3D effects require perspective**: Without `perspective` and `transform-style: preserve-3d`, 3D transforms just look like squash/stretch. Z-axis rotation creates depth; combine with animations for engaging effects using pure CSS.
