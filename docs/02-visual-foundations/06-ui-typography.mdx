## 핵심 철학

**파레토 법칙의 적용**: 대부분의 UI는 텍스트와 버튼, 그리고 일부 아이콘으로 구성됩니다. 타이포그래피를 마스터하는 것은 20%의 노력으로 80%의 결과를 얻는 지름길입니다.

### 타이포그래피가 UI에 미치는 영향

원본 디자인에서 모든 폰트 스타일을 제거하면:
- 선택된 탭을 구분할 수 없음 (사용자가 현재 위치를 파악 불가)
- 버튼의 역할이 불명확
- 피드에 계층 구조와 집중 포인트가 없어 정보 파악 곤란
- 모든 요소가 하나의 텍스트 덩어리로 보임

---

## 1. 시각적 계층 구조 만들기 (Visual Hierarchy)

### 1.1 기본 원칙: 크기와 굵기

**핵심 인사이트**: 제목을 강조하는 가장 기본적인 방법은 크기와 굵기입니다.

### 1.2 요소 분리를 위한 4가지 속성

인간의 뇌가 패턴과 형태를 인식하는 방식을 활용:

| 속성 | 용도 | 효과 |
|------|------|------|
| **크기 (Size)** | 중요도 강조 | 혼동 가능 (3개 그룹으로 보일 수 있음) |
| **색상 (Color)** | 강조/비강조 | 단독 사용 시 불분명 |
| **간격 (Spacing)** | 그룹화/분리 (근접성의 법칙) | **가장 효과적** |
| **굵기 (Weight)** | 시각적 무게감 | 크기와 결합 시 강력 |

**핵심 인사이트**: 이 4가지 속성을 조합하면 요소를 그룹화하거나 분리할 수 있습니다.

---

## 2. 색상을 통한 강조 전략 (HSL Color System)

### 2.1 HSL 색상 모델의 이해

HSL = Hue (색조) + Saturation (채도) + Lightness (명도)

| 속성 | 범위 | 설명 |
|------|------|------|
| **Hue (H)** | 0-360° | 색조 (0°=빨강, 120°=초록, 240°=파랑) |
| **Saturation (S)** | 0-100% | 채도 (100%=가장 진함, 0%=회색) |
| **Lightness (L)** | 0-100% | 명도 (0%=검정, 50%=기본색, 100%=흰색) |

### 2.2 대비를 통한 강조 기법

**핵심 전략**: 흰색 배경에 검은 텍스트는 이미 최대 대비입니다. 특정 요소를 강조하려면 **다른 요소를 비강조**해야 합니다.

### 2.3 명도 값 선택 가이드 (다크 모드)

| 요소 유형 | Lightness 값 | 용도 |
|-----------|-------------|------|
| 최우선 텍스트 (제목) | 90-100% | 가장 중요한 정보 |
| 본문 텍스트 | 60-70% | 읽기 가능하면서 강조 안 함 |
| 보조 정보 (메타데이터) | 40-50% | 덜 중요한 정보 |
| 배경 카드 | 10-20% | 콘텐츠와 구분 |
| 비활성 상태 | 30-40% | 사용 불가 표시 |

---

## 3. 최소 폰트 크기 시스템 (Minimal Type Scale)

### 3.1 "99% 법칙" - 3개 크기면 충분하다

**핵심 인사이트**: 대부분의 UI는 3-4개의 폰트 크기만으로 충분합니다.

YouTube 인터페이스 분석:
- **14px**: 본문, 버튼, 설명, 댓글, 메타데이터 (전체의 90%)
- **16px**: 제목
- **20px**: 채널 이름 (강조)

### 3.2 굵기와 색상으로 크기 대신하기

**놀라운 사실**: 같은 16px 폰트도 굵기와 색상만 바꾸면 완전히 다른 시각적 무게감을 가집니다.

### 3.3 실전 타입 스케일 구축법

1. 기본 폰트 크기 선택: 14px 또는 16px
2. 기본 굵기: 400 (Regular)
3. 기본 색상: 100% lightness (다크 모드)
4. 필요할 때만 ±2px 조정

---

## 4. 줄 높이 (Line Height) 마스터하기

### 4.1 줄 높이의 숨겨진 역할

**핵심**: 줄 높이는 자동으로 하단 마진 역할을 합니다. 대부분의 경우 수동으로 margin-bottom을 추가할 필요가 없습니다.

### 4.2 줄 높이 설정 가이드

| 폰트 크기 | 권장 line-height | 비고 |
|-----------|------------------|------|
| 제목 (20px+) | 1.2 - 1.3 | 타이트하게 |
| 본문 (14-18px) | 1.5 - 1.6 | 가독성 우선 |
| 긴 텍스트 | 1.75 - 2.0 | 여유롭게 |

---

## 5. 다크 모드 ↔ 라이트 모드 전환

### 5.1 간단한 수학 공식

라이트 모드 명도 = 100 - 다크 모드 명도

| 다크 모드 | 라이트 모드 | 용도 |
|-----------|-------------|------|
| hsl(0, 0%, 0%) | hsl(0, 0%, 100%) | 배경 |
| hsl(0, 0%, 10%) | hsl(0, 0%, 90%) | 카드 배경 |
| hsl(0, 0%, 60%) | hsl(0, 0%, 40%) | 보조 텍스트 |
| hsl(0, 0%, 100%) | hsl(0, 0%, 0%) | 주요 텍스트 |

---

## 6. 시각적 계층 vs 문서 계층

### 6.1 중요한 원칙

코드는 문서 구조를 따르고 (Semantic HTML)
스타일은 시각적 계층을 따른다

**나쁜 예시**: 모든 H1이 같은 스타일
**좋은 예시**: 문서 구조는 유지하되, 스타일은 컨텍스트에 맞게 조정

## 7. 동적 콘텐츠 대응 (Responsive Typography)

### 7.1 clamp()로 유동적 크기 조절

```css
:root {
  /* clamp(최소값, 선호값, 최대값) */
  --heading-size: clamp(1.5rem, 1rem + 2.5vw, 3rem);
  --body-size: clamp(0.875rem, 0.8rem + 0.25vw, 1rem);
}

/* 긴 숫자/텍스트 처리 */
.stat-number {
  font-size: clamp(1rem, 0.5rem + 2vw, 2.5rem);
  font-variant-numeric: tabular-nums; /* 숫자 폭 균일화 */
}

/* Container Query 활용 */
@container (max-width: 300px) {
  .stat-number {
    font-size: var(--text-sm);
  }
}
```

### 7.2 React Text 컴포넌트

```tsx title="Text.tsx"
import React from 'react';

type TextVariant = 'heading' | 'subheading' | 'body' | 'caption' | 'overline';
type TextColor = 'primary' | 'secondary' | 'muted' | 'error' | 'success';

interface TextProps {
  variant?: TextVariant;
  color?: TextColor;
  as?: keyof JSX.IntrinsicElements;
  children: React.ReactNode;
}

const variantStyles: Record<TextVariant, React.CSSProperties> = {
  heading: { fontSize: 'var(--text-2xl)', fontWeight: 700, lineHeight: 1.2 },
  subheading: { fontSize: 'var(--text-lg)', fontWeight: 600, lineHeight: 1.3 },
  body: { fontSize: 'var(--text-base)', fontWeight: 400, lineHeight: 1.6 },
  caption: { fontSize: 'var(--text-sm)', fontWeight: 400, lineHeight: 1.5 },
  overline: {
    fontSize: 'var(--text-xs)', fontWeight: 600, lineHeight: 1.5,
    letterSpacing: '0.05em', textTransform: 'uppercase' as const,
  },
};

const colorMap: Record<TextColor, string> = {
  primary: 'var(--text-primary)',
  secondary: 'var(--text-secondary)',
  muted: 'var(--text-muted)',
  error: 'var(--color-error)',
  success: 'var(--color-success)',
};

export function Text({
  variant = 'body', color = 'primary', as: Tag = 'p', children,
}: TextProps) {
  return (
    <Tag style={{ ...variantStyles[variant], color: colorMap[color], margin: 0 }}>
      {children}
    </Tag>
  );
}

// Usage:
// <Text variant="heading" as="h1">Page Title</Text>
// <Text variant="caption" color="muted">Last updated 2 hours ago</Text>
```

## 8. 접근성 (Accessibility) 고려사항

### 8.1 Rem 단위 사용 이유

사용자가 브라우저 폰트 크기를 조정할 수 있도록 rem 단위 사용

### 8.2 색상 대비 비율

WCAG AA 기준:
- 일반 텍스트: 4.5:1 이상
- 큰 텍스트 (18px+): 3:1 이상

## 9. 실전 체크리스트

### 디자인 시작 전
- [ ] 기본 폰트 크기 결정 (14px 또는 16px)
- [ ] 폰트 패밀리 선택 (Google Fonts 등)
- [ ] 타입 스케일 정의 (3-5개 크기)
- [ ] 색상 변수 설정 (HSL 사용)

### 컴포넌트 디자인 중
- [ ] 제목이 본문과 분리되어 보이는가?
- [ ] 중요한 정보가 강조되었는가?
- [ ] 덜 중요한 정보는 비강조되었는가?
- [ ] 줄 높이가 적절한가? (수동 마진 필요 없음)

### 디자인 완료 후
- [ ] 전체 UI를 축소해서 계층 구조 확인
- [ ] 동적 콘텐츠 (긴 텍스트/숫자) 테스트
- [ ] 다크/라이트 모드 모두 확인
- [ ] 색상 대비 비율 검증 (접근성)
- [ ] 다양한 화면 크기에서 테스트

### 사용자 테스트
- [ ] 실제 사용자에게 보여주기
- [ ] 동료에게 피드백 받기
- [ ] "어디를 먼저 보게 되나요?" 질문하기

- Adam Wathan & Steve Schoger, "Refactoring UI" (2018)

> **실제 사례**: YouTube 인터페이스는 전체의 90%가 14px 폰트 하나만 사용하며, 굵기(weight)와 색상(color)만으로 제목/본문/메타데이터를 구분합니다. 이는 "최소 폰트 크기 시스템"의 대표적 사례로, 일관성과 성능(폰트 로딩)을 동시에 확보합니다.

> **심화 이론**: HSL 색상 모델을 타이포그래피에 활용하는 것은 **명도 기반 계층(Lightness-based Hierarchy)** 전략입니다. 흰색 배경에 검은 텍스트는 이미 최대 대비이므로, 특정 요소를 강조하려면 다른 요소를 명도 60-70%로 낮춰(de-emphasize) 상대적 차이를 만듭니다. 이는 절대값이 아닌 **상대적 지각**을 이용한 기법입니다.

> **심화 이론**: Line-height가 자동 마진 역할을 하는 이유는 **CSS Box Model의 Line Box** 개념 때문입니다. Line-height는 텍스트 위아래에 균등하게 분배되어 "Half-leading"을 형성하므로, 연속된 텍스트는 자연스러운 수직 간격을 가집니다. 이를 이해하면 불필요한 `margin-bottom` 추가를 피할 수 있습니다.
