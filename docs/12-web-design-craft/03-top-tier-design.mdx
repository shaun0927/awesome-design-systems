## 디자인 시스템 효율화 & 노하우 정리

> 출처: "The Easy Way to Design Top Tier Websites" 영상 요약
> 핵심 관점: **디자인 시스템 구축을 통한 효율화**, 실전 노하우, 창의적 프로세스

---

## 핵심 철학

> **창의성은 '순간'이 아니라 '프로세스'다.**
> 새로운 디자인을 백지에서 만들 필요 없다. 최고의 디자이너들은 이미 존재하는 요소들을 **고유한 방식으로 조합**한다.

---

## Rule 1: 좋은 디자인 = 최소한의 디자인 (Less is More)

### 원칙
- 핵심 기능에 집중하여 더 좋고 유용하게 만들 것
- 색상, 텍스트, 시각적 잡음(clutter)을 줄일 것

### 실전 노하우
| 흔한 실수 | 올바른 접근 |
|-----------|------------|
| 헤더부터 시작해서 아래로 내려가기 | **핵심 기능/셀링 포인트**부터 시작 |
| "섹션 몇 개?", "버튼 디자인은?", "너비는?" 등 HOW 질문 | **"이 웹사이트의 핵심 기능은?"** WHAT 질문 |
| 요소를 계속 추가 | 최소한의 요소(제목 + 입력 필드 + 버튼)로 시작 |

**핵심**: 대부분의 웹사이트는 **제목(heading) + 입력 필드(input) + 버튼(button)** 만으로 충분하다. 거기서 시작하라.

---

## Rule 2: 유사성과 근접성의 법칙 (Gestalt Theory 활용)

### 원칙
- **유사성 법칙**: 형태(shape), 크기(size), 색상(color)이 같은 요소는 같은 그룹으로 인식
- **근접성 법칙**: 가까이 있는 요소는 관련 있는 것으로 인식
- 뇌는 정보를 **전체(whole)로 먼저 처리**한 후 세부 사항을 인식

### 디자인 시스템 효율화 포인트
- 유사성 법칙을 적용하면 **디자인 일관성**이 자동으로 확보됨
- **구현도 쉬워짐** → 반복 패턴이 생기므로 컴포넌트화에 유리
- 근접성 법칙은 **레이아웃과 간격(spacing) 설계**의 기본 원리

### 핵심 기준
> 디자인은 **몇 초 안에 스캔 가능(scannable)**해야 한다.

---

## Rule 3: 간격(Spacing)은 생각보다 훨씬 많이 필요하다

### 원칙
- 개별 요소에 집중할 때는 간격이 과해 보이지만, 사용자는 **전체 UI를 먼저 스캔**함
- **많은 간격에서 시작 → 줄여나가기** (반대가 아님!)

### 실전 프로세스
```
1. 넉넉한 간격으로 시작 (예: 40px)
2. 전체 디자인을 한 눈에 확인
3. 만족할 때까지 간격을 줄여나감
```

> 이 작업을 수동으로 반복하면 지루하고 비효율적 → **디자인 시스템이 필요한 이유**

---

## Rule 4: 디자인 시스템 구축 (핵심 효율화 전략)

### 왜 필요한가?
- 수동 조정의 반복을 **체계적 선택**으로 전환
- 크고 복잡한 웹사이트/앱일수록 필수
- 기본 원리를 이해하면 **CSS 프레임워크 없이도** 충분히 스타일링 가능

### 1) 간격 시스템 (Spacing System)

**핵심 규칙: 4의 배수로 값 설정**

| px 값 | rem 값 (÷16) | 용도 예시 |
|--------|-------------|-----------|
| 4px | 0.25rem | 아이콘 내부 패딩 |
| 8px | 0.5rem | 인라인 요소 간격 |
| 12px | 0.75rem | 관련 요소 간 간격 |
| 16px | 1rem | 기본 간격 |
| 20px | 1.25rem | 섹션 내부 간격 |
| 24px | 1.5rem | 카드 패딩 |
| 32px | 2rem | 섹션 간 간격 |
| 40px | 2.5rem | 대형 섹션 간격 |
| 48px | 3rem | 페이지 레벨 간격 |

**효율화 팁:**
- `rem` 단위 사용 → 사용자 시스템 설정에 반응하는 적응형 디자인
- **CSS 변수(Custom Properties)로 정의** → 값 변경 시 전체 반영
- Lorem Ipsum이나 모호한 더미 데이터로 디자인하지 말 것 → **실제 데이터 기반**으로 간격 결정

```css
:root {
  --space-xs: 0.25rem;   /* 4px */
  --space-sm: 0.5rem;    /* 8px */
  --space-md: 1rem;      /* 16px */
  --space-lg: 1.5rem;    /* 24px */
  --space-xl: 2.5rem;    /* 40px */
  --space-2xl: 3rem;     /* 48px */
}
```

### 2) 타이포그래피 시스템 (Font System)

- **폰트 1개**와 프로젝트에 맞는 **타입 스케일(type scale)** 선택
- CSS 변수로 글로벌 정의

**Line Height 핵심 규칙:**

> **줄 높이(line-height)는 폰트 크기에 반비례한다.**

| 폰트 크기 | 권장 line-height |
|-----------|-----------------|
| 작은 텍스트 (14px 이하) | 1.6 ~ 1.8 (넉넉하게) |
| 본문 (16px) | 1.5 ~ 1.6 |
| 큰 제목 (24px+) | 1.2 ~ 1.3 |

**효율화 팁**: 적절한 line-height는 **텍스트 요소 간 margin-top 역할**도 함 → 별도 spacing 설정 불필요

### 3) 색상 시스템 (Color System)

**최소 구성 (4색이면 충분):**

| 역할 | 설명 |
|------|------|
| 다크 텍스트 색 | 본문, 제목용 |
| 라이트 배경 색 | 배경, 카드용 |
| 액센트 색 1 | 브랜드 컬러, CTA |
| 액센트 색 2 | 보조 강조, 호버 |

**주의사항:**
- 색상 심리학 과대평가 금지 → 실용적으로 선택
- **가독성(legibility)** 최우선
- 사용자를 압도하지 않을 것
- `text-align: center`는 **문단(paragraph)이나 작은 텍스트에 사용 금지** (가독성 저하)

### 4) 컴포넌트 시스템 (Component System)

**최소한으로 시작:**
- 링크(Link): Primary + Secondary
- 버튼(Button): Primary + Secondary

> 이 4가지만 정의하면 디자인 시스템의 기본 골격 완성

## Rule 5: 계층 구조(Hierarchy)가 전부다

### 원칙
- 사용자가 **탐색하고 중요한 액션을 찾도록** 특정 요소를 강조
- 강조 도구: **크기(size)**, **두께(weight)**, **색상(color)**

### 강조 프로세스 (실전 순서)

```
1단계: "사용자가 가장 먼저 찾을 것은?" 질문
       → 예: 제목(title)

2단계: 부차적 정보의 대비(contrast) 낮추기
       → 강조하려면 다른 요소를 "약화"시키는 것이 효과적

3단계: 폰트 굵기(weight) 증가
       → 색상만으로 부족할 때

4단계: 폰트 크기(size) 증가
       → 추가 강조 필요 시

5단계: 줌 아웃하여 전체 확인
       → 핵심 정보가 스캔 가능한지 검증
```

### 핵심 인사이트
| 포인트 | 설명 |
|--------|------|
| **강조 = 약화** | 무언가를 강조하려면 경쟁 요소를 약화시켜야 함 |
| **컨텍스트 의존** | 같은 `<h1>`도 상황에 따라 크기/마진이 달라야 함 |
| **태그 ≠ 스타일** | `<h3>`가 `<h2>`보다 클 수 있음 → 의미론과 시각적 계층은 별개 |
| **과도한 디자인 금지** | 디자인을 더 할수록 대부분 더 못생겨짐 |

## 깊이감(Depth) 추가 기법

> Rule 1~5를 지킨 뒤, 선택적으로 적용하는 **마감 디테일**

| 기법 | 활용 |
|------|------|
| **그림자(Shadow)** | 중요 요소 부각, 실선 테두리 대체 |
| **액센트 컬러** | 중요 요소 하이라이트 |
| **그라데이션** | 단색 → 미묘한 그라데이션으로 활력 추가 |
| **카드(Card)** | 밋밋한 리스트/테이블을 카드 형태로 변환 |

**핵심 원리**: 사용자에게 가깝게 느껴지는 요소일수록 더 많은 시선을 끈다.

## 창의적 프로세스 5단계 (실전 워크플로우)

### Step 1: 기본기 학습
- 위의 Rule 1~5 숙지
- 관련 서적 학습

### Step 2: 영감 수집 (Inspiration Gathering)
- Top Tier 웹사이트 분석 및 스타일 연구
- **Mobbin**, Figma Community 등 활용
- 필터링: 업종(카테고리) + 섹션 유형으로 검색
- 마음에 드는 디자인을 라이브러리에 저장

### Step 3: 분석 및 아이디어 도출
- 수집한 영감을 **사용자 관점**에서 분석
- 좋아하는 요소를 메모
- 예시 (금융 앱 후기 섹션):
  - 단순하고 고유한 디자인
  - 실제 사람 얼굴 포함
  - 간결한 언어
  - 감정이 느껴지는 리뷰 (제네릭한 나열 X)

### Step 4: 의도적 휴식 (Incubation)
> **아이디어가 떠올랐어도 바로 디자인하지 말 것!**

- 문제에서 벗어나 다른 활동
- 잠재의식이 아이디어를 숙성시킴
- 돌아오면 자연스럽게 새로운 아이디어 발생
- 안 되면? → **스트레스/수면 부족** 해결 먼저

### Step 5: 완성에 집착하지 않기
- 자신의 디자인에 **사랑에 빠지지 말 것**
- 동료/친구에게 피드백 요청
- 사용자 테스트 수행
- 피드백 기반 조정에 열린 자세
- **여러 번 디자인**해야 하나의 좋은 섹션이 나올 수 있음

> **"완벽한 계획보다 불완전한 실행이 낫다. 일단 뭐라도 만들어라."**

## 종합 정리: 디자인 시스템 효율화 체크리스트

```
□ Spacing System: 4의 배수 기반 CSS 변수 정의
□ Typography: 폰트 1개 + 타입 스케일 + line-height 규칙
□ Color Palette: 4색 최소 구성 (다크/라이트/액센트1/액센트2)
□ Components: Link(2종) + Button(2종) 기본 세트
□ Hierarchy: 크기/두께/색상으로 시각적 우선순위 설정
□ Scanability: 줌 아웃 테스트로 핵심 정보 가시성 확인
□ Inspiration: 레퍼런스 수집 → 분석 → 휴식 → 실행 사이클
□ Feedback Loop: 완성 후 피드백 → 조정 반복
```

## AlphaView 프로젝트 적용 포인트

| 원칙 | AlphaView 적용 |
|------|----------------|
| 최소 디자인 | 뉴스 카드/모달에서 핵심 정보(제목, 티커, 점수)만 강조 |
| Spacing System | Tailwind의 spacing scale 활용 (이미 4의 배수 기반) |
| 계층 구조 | Alpha Score, 제목, 날짜 순으로 시각적 우선순위 |
| 색상 시스템 | 브랜드 액센트 컬러로 CTA/스코어 하이라이트 |
| 카드 패턴 | 뉴스 목록을 카드 컴포넌트로 통일 |
| 스캔 가능성 | 뉴스 피드의 빠른 스캔을 위한 일관된 레이아웃 |
